# 项目要求
服务器端通过Springboot+SpringDataJPA，实现学生选课管理系统，功能如下：

1）前端技术不限（禁止使用JSP），界面美观，操作友好。

2）学生注册功能，输入姓名（必填）、年龄（必填）、手机号（不能重复，必填）、入学时间（必填）、下拉框选择专业（必选）。系统自动生成学号（学号按专业编号+入学年份+序号的规律生成，同一个专业和年级不能重复序号。）

3）学生注册成功后，系统会返回登陆页面，同时把生成的学号填入到学生登录系统的账号输入框，并生成默认密码“123456”。

4）学生首次登陆，需要采用学号和初始密码登录。登陆后，系统首先提示修改密码，没有修改密码不能继续操作。新密码必须由数字、字母、字符，且大于7位组成。系统验证生成的密码是否符合标准。密码修改成功后，出现选课功能的菜单。下次登陆时，能直接进入选课功能。

5）学生选课功能，学生登录后，能直接看到已选课程列表和未选课程列表。也能能看到已选课程的学分总和和未选课程的学分总和。学生可以通过点击未选课程列表中每个课程的选修按钮实现选课，课程能自动添加到已选课程的列表。也能通过点击已选课程的退选按钮实现课程退选。已选课程学分总和和未选课程学分总和自动修改。

6）课程信息管理功能，包括课程名字、编号、简介、学分，四个属性。课程信息管理模块，只有管理员角色的账号可以操作。注意选课与课程外键管理，修改课程信息会影响选课的信息。能实现课程信息的增删改查的操作，但学生已选课程不能删除，删除时并给出提示。

7）专业信息管理功能，包括专业名字、编号、简介三个属性。专业信息管理模块，只有管理员角色的账号可以操作。注意学生专业属性与专业外键管理，修改专业信息会影响学生的专业信息。能实现专业信息的增删改查，有学生的专业不能删除，，删除时并给出提示。

8）账号信息管理功能，包括账号名字、密码（md5加密），角色三个属性。账号信息管理模块，只有管理员角色的账号可以操作。

9）选课信息查询功能，能看到所有学生的选课情况。并能实现各种查询功能，由管理员操作。

10）把系统部署在笔记本电脑，不用开发环境演示。

11）使用MySQL 8.0数据库。
